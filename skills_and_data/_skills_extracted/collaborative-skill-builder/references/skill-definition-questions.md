# Skill Definition Questions Guide

This guide provides a structured set of questions designed to facilitate the collaborative definition of new skills. By engaging the user with these inquiries, the agent can effectively gather concrete examples, understand underlying needs, and ensure the developed skill is comprehensive and tailored.

## Understanding the Core Purpose

To begin, open-ended questions are used to grasp the fundamental intent behind the skill request. This initial phase aims to establish a broad understanding of the skill's primary function and the general problems it seeks to address.

| Question Category | Example Questions |
| :--- | :--- |
| **Purpose & Scope** | "What should this skill do? Please describe its purpose and the specific tasks or workflows it should handle."<br>"What problem are you trying to solve with this skill?"<br>"Can you give me a high-level overview of the desired outcome?" |

## Clarifying Scope and Functionalities

Once the core purpose is understood, more specific questions are posed to refine the skill's scope and identify its key functionalities. This stage delves into the methodologies, analytical capabilities, and creative processes the skill should support.

| Aspect | Example Questions |
| :--- | :--- |
| **Frameworks/Methodologies** | "Should it focus on specific frameworks or methodologies (e.g., Design Thinking, TRIZ, Agile)?" |
| **Analysis Type** | "Do you want it to help with a particular type of analysis (e.g., market research, patent analysis, technical feasibility)?" |
| **Ideation vs. Evaluation** | "Should it assist in generating creative ideas, or more in structuring and evaluating them?" |
| **Industry Focus** | "Are there any specific industries or domains this skill should cater to (e.g., Pharmaceuticals, Medical Devices, Tech, Consumer Goods)?" |
| **Process Flow** | "What are the key steps or phases involved in the process you envision for this skill?" |
| **Inputs & Outputs** | "What kind of inputs will this skill typically receive, and what kind of outputs should it produce?" |

## Implementation and Regulatory Considerations

For skills involving product development, implementation, or market entry, it is crucial to address practical and regulatory aspects. These questions ensure that the skill incorporates necessary compliance, operational, and technical details.

| Consideration | Example Questions |
| :--- | :--- |
| **Regulatory Compliance** | "Are there specific regulatory bodies or standards that need to be considered (e.g., FDA, EMA, cGMP, ISO)?" |
| **Target Markets** | "Are there specific target markets or regions you're most interested in (e.g., USA, EU, China, GCC)?" |
| **Project Management** | "Should the skill help with project management aspects (e.g., Gantt charts, milestones) or more on the technical/documentation side?" |
| **Reference Materials** | "Do you have any specific 'Gold Standard' references, internal templates, or existing protocols you'd like me to incorporate into the skill's logic?" |
| **Facility & Operations** | "What are the utility requirements, waste management considerations, or production suitability aspects that need to be addressed?" |
| **Advanced Technologies** | "Are there any advanced technologies or specific manufacturing protocols (e.g., SFE, lyophilization, spray drying) that should be included?" |

## Iterative Refinement and Feedback

Throughout the development process, maintaining an iterative feedback loop is essential for refining the skill and ensuring it meets user expectations. These questions facilitate continuous improvement.

| Feedback Prompt | Purpose |
| :--- | :--- |
| "Does this cover everything you need, or should I add/modify anything?" | To solicit comprehensive feedback on the current draft. |
| "Would you like me to go into more detail in any of these specific areas?" | To identify areas requiring further elaboration or specificity. |
| "Is there anything unclear or that needs further explanation in the drafted content?" | To pinpoint ambiguities or areas of confusion. |

By systematically addressing these questions, the `collaborative-skill-builder` skill can effectively guide the creation of new, highly tailored capabilities, transforming initial user requests into robust and functional tools.
